# Contributing

Спасибо за вклад в CorgPhish.

## Перед PR
1. Создайте отдельную ветку от актуальной `main` или рабочей feature-ветки.
2. Внесите изменения с понятными коммит-сообщениями.
3. Проверьте сборку локально:
```bash
./scripts/verify.sh
./scripts/package.sh
```

## Требования к Pull Request
- Опишите, что и зачем изменено.
- Укажите возможные риски и обратную совместимость.
- Добавьте шаги ручной проверки.
- Обновите документацию, если меняется поведение.

## Стиль изменений
- Не добавляйте секреты, токены, приватные данные.
- Держите изменения целевыми, без лишних рефакторингов.
- Для крупных правок добавляйте краткое пояснение в PR.

## Сообщение о проблемах
- Для ошибок используйте шаблон `Bug report`.
- Для идей и улучшений используйте шаблон `Feature request`.
