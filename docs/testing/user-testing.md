# Пользовательское тестирование

В основной пользовательской выборке участвовали `70` человек разного возраста.

## Состав основной выборки
1. Подростки и молодые пользователи — `30` человек.
2. Пользователи 30–50 лет — `20` человек.
3. Пользователи 70+ — `20` человек.

## Как проводилось
- Участникам давались безопасные, подозрительные и фишинговые сценарии.
- Проверялась реакция на popup, blocked page, предупреждения на странице и кнопки действий.
- Отдельно оценивались понятность вердикта, заметность предупреждения и вероятность ошибочного действия.

## Что показало тестирование
1. Пользователи в большинстве случаев правильно различали сценарии `safe`, `warning` и `block`.
2. Экран блокировки заметно снижал риск продолжения ввода данных на опасной странице.
3. Наиболее чувствительным сценарием оказались сокращённые ссылки и поздние редиректы.
4. Старшая возрастная группа особенно хорошо воспринимала явный сценарий с отдельным экраном блокировки.
5. Для части пользователей среднего возраста требовалась более быстрая настройка исключений прямо из popup.

## Практический вывод
Пользовательское тестирование подтвердило не только техническую корректность механизма предупреждения, но и его пригодность для реального использования без специальной подготовки.
