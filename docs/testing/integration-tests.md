# Интеграционные тесты

Интеграционные тесты выполнялись на release-сборке без подмены логики.

## Что фиксировалось в каждом запуске
1. Итоговый вердикт (`safe`, `warning`, `block`).
2. Факт и момент срабатывания защитного действия.
3. Время до вердикта.
4. Источник решения (`rules`, `ml`, `fallback`).

## Суммарный объём
- Всего интеграционных запусков: `348`

## Непрошедшие тесты первого прогона
### INT-42 — «Переход по сокращённой ссылке и позднее срабатывание проверки»
- Ожидалось: предупреждение до открытия целевой формы.
- Получено: `2` сбоя в `12` прогонах, вердикт приходил с опозданием.
- Причина: задержка в цепочке `редирект -> догрузка -> эвристика` в одном браузере.
- Статус: требуется доработка обработки многошаговых редиректов.

## Интерпретация результатов
- Контур правил и ML показывает высокую полноту по фишинговым кейсам.
- Fallback остаётся устойчивым даже при полном отказе ML.
- Основная зона дальнейшего усиления — скорость реакции на часть редирект-цепочек.
