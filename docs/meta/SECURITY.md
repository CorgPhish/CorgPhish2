# Security Policy / Политика безопасности

## Reporting a vulnerability / Сообщить об уязвимости
If you believe you have found a security issue, please report it privately.
Если вы нашли уязвимость, пожалуйста, сообщите о ней приватно.

Preferred channel / Предпочтительный канал:
- GitHub Security Advisories (if enabled).

Fallback / Если недоступно:
- Open a GitHub issue with the prefix `[SECURITY]` and avoid sensitive details.

## What to include / Что приложить
- Clear reproduction steps.
- Expected vs actual behavior.
- Affected version and environment.

- Чёткие шаги воспроизведения.
- Ожидаемое и фактическое поведение.
- Версию и окружение.

## Response / Ответ
We will acknowledge receipt and work on a fix as soon as possible.
Мы подтвердим получение и постараемся исправить проблему как можно быстрее.
